MERGE في SQL هي أمر بنستخدمه لما نكون عايزين نحدّث البيانات في جدول اعتمادًا على بيانات موجودة في جدول تاني.
الميزة في MERGE إنها بتخلينا نعمل أكتر من عملية في نفس الوقت زي INSERT (إضافة بيانات جديدة) أو UPDATE (تعديل بيانات موجودة) أو حتى DELETE (حذف بيانات) باستخدام Query واحدة بس.

يعني مثلًا لو عندنا جدول اسمه Employees فيه بيانات الموظفين، وعندنا جدول تاني فيه بيانات جديدة أو محدثة لنفس الموظفين. بدل ما نكتب كذا أمر SQL عشان نضيف أو نعدل البيانات، ممكن نستخدم MERGE عشان نقارن بين الجدولين.

لو السجل موجود في الجدولين يتم تحديثه (UPDATE).
لو السجل موجود في الجدول الجديد بس يتم إضافته (INSERT).
ولو السجل موجود في الجدول القديم بس ومش موجود في الجديد ممكن نحذفه (DELETE) لو عايزين.

ببساطة MERGE بتوفر علينا وقت ومجهود، لأنها بتجمع كذا عملية في أمر واحد وبتستخدم كتير في الحالات اللي بنحتاج فيها نجمع او نقارن البيانات بين جدولين لنفس المعلومات .
مثال عليها:-
Employees
| ID | Name  |
| -- | ----- |
| 1  | Ahmed |
| 2  | Sara  |
| 3  | Omar  |
NewEmployees
| ID | Name         |
| -- | ------------ |
| 2  | Sara Mohamed |
| 3  | Omar         |
| 4  | Ali          |
دلوقتي عايزين نحدّث جدول Employees بالبيانات الجديدة.
MERGE Employees AS T
USING NewEmployees AS S
ON T.ID = S.ID
WHEN MATCHED THEN
UPDATE SET T.Name = S.Name
WHEN NOT MATCHED THEN
INSERT (ID, Name)
VALUES (S.ID, S.Name);

Result
| ID | Name         |
| -- | ------------ |
| 1  | Ahmed        |
| 2  | Sara Mohamed |
| 3  | Omar         |
| 4  | Ali          |
بس كد ف بدل منعمل Insert او Update او Delete حتي كل وحده نكتبلها Query لوحدها ممكن نعمل MERGE بتوفر وقت و مجهود و Code منظم ^_^
